I PARTE. Análisis DAFO (IDE PYTHON vs Lenguaje C)

(Valor: 10 puntos)

El análisis DAFO se realiza comparando las fortalezas y debilidades del ecosistema de desarrollo de **Python** (incluyendo sus IDEs modernos) versus el lenguaje **C** tradicional.

Categoría IDE PYTHON Lenguaje C
Fortalezas (F) Ecosistema de librerías (IA, Data Science, Web). Sintaxis legible, curva de aprendizaje rápida. Excelente para scripting y automatización. Alto rendimiento y velocidad. Control directo sobre la memoria y hardware. Fundamental para entender sistemas operativos.
Oportunidades (O) Crecimiento masivo en IA/ML. Demanda empresarial y en startups. Desarrollo rápido de prototipos. Relevancia en sistemas embebidos (IoT) y programación de bajo nivel. Oportunidad en optimización de rendimiento.
Debilidades (D) Velocidad de ejecución más lenta que C (por ser interpretado). Gestión de dependencias (pip) compleja a veces. No apto para control estricto de recursos. Curva de aprendizaje empinada (punteros, memoria manual). Menos librerías de alto nivel para tareas complejas modernas.
Amenazas (A) Competencia de lenguajes rápidos (Go, Julia). Riesgo de dependencia excesiva de librerías de terceros. C++ y Rust ofrecen mayor seguridad de memoria sin sacrificar tanto rendimiento.

II PARTE. CASO DE ESTUDIO 1 (Servidor de Datos Linux)

(Valor: 30 puntos)

1. Requerimiento

Implementación de un Servidor de Datos (LINUX) accesible vía SSH y FTP para respaldar y guardar Algoritmos de la Universidad X desde un Equipo X (Laptop).

2. Procedimiento y Configuración

El proyecto se implementó usando una Máquina Virtual con Ubuntu Server.

2.1 Configuración de Máquina Virtual (VM)

  • **Software de Virtualización:** VirtualBox.
  • **Sistema Operativo:** Ubuntu Server (LINUX).
  • **Configuración de Red:** Adaptador Puente (Bridge Adapter) para asegurar que la VM obtenga una IP en la red local del Equipo X (Laptop).

Imagen 2. Configuración de red de la VM en modo Adaptador Puente (Bridge) para conectividad directa.

2.2 Servicios de Servidor de Datos

Servicio Paquete Linux Propósito
**SSH** openssh-server Administración remota segura y ejecución de comandos.
**FTP** vsftpd Transferencia eficiente de archivos (algoritmos) para respaldo.

III PARTE. CASO DE ESTUDIO 2 (Esquema de Red LAN)

(Valor: 30 puntos)

1. Requerimiento

Diseño de esquema de RED LAN para la administración del Centro Regional David-Chiriquí, cubriendo todos los espacios (A-O) con conectividad cableada e inalámbrica.

2. Requerimientos Técnicos (Colegio AB)

EQUIPOS CANTIDAD IP/MÁSCARA/PE/DNS OBSERVACIÓN
Impresoras 2 DHCP/Fija (opcional)
Servidor de Datos 1 IP Fija: 10.10.10.10
Laptop 6 DHCP (Inalámbrico)
PC 3 DHCP (Cableado)
Switch (2960) 1 (24 puertos) - Punto de agregación.
Router (300N) 1 PE: 10.10.10.1 Gateway y DHCP Server.
Extensor de Señal 2 - Cobertura inalámbrica.
Subred - MR: 255.255.255.0 Subred Clase A, /24.
DNS - DNS1: 8.8.8.8, DNS2: 8.8.4.4

3. Topología de Red LAN (Diseño en Cisco Packet Tracer)

  • **Conectividad Central:** El Router 300N (10.10.10.1) es el punto central de la red y el servidor DHCP.
  • **Cableado Fijo:** El Switch 2960 conecta de manera robusta y por cable al Servidor de Datos (10.10.10.10), PCs e Impresoras.
  • **Cobertura Total (Inalámbrica):** El Router junto con los 2 Extensores de Señal garantizan que las 6 Laptops tengan acceso Wi-Fi desde cualquier espacio (A-O).
