Resumen Ejecutivo

Este informe presenta el desarrollo del **Sistema de Información para CHINOS CAFE**, un proyecto que abarca la configuración de infraestructura, el diseño de red, la lógica algorítmica y la implementación en Python. El objetivo es ofrecer una solución centralizada y robusta para la gestión de datos de los establecimientos, permitiendo la comunicación efectiva entre las sucursales y el servidor central de **Linux con MySQL**.

I. Laboratorio de Servidor (Linux + MySQL)

La implementación del Servidor de Datos se realizó en una Máquina Virtual utilizando **Ubuntu Server 22.04 LTS**. Esta elección garantiza un entorno robusto y estable para el gestor de bases de datos MySQL, que actúa como el repositorio centralizado de la información del negocio.

1.1. Configuración de la Máquina Virtual

La VM se configuró con **2GB de RAM, 2 CPUs virtuales** y una interfaz de red en modo **Adaptador Puente (Bridge)** para asegurar que obtenga una dirección IP dentro de la subred local (192.168.1.x) y sea accesible por los clientes de la red física.

Captura de la configuración de la VM (Adaptador Puente)

1.2. Instalación y Configuración de MySQL Server

La instalación de MySQL se realizó mediante `sudo apt install mysql-server`. Se creó un usuario dedicado (`chinos_user`) con permisos específicos para acceder al esquema `chinos_cafe` y se configuró el servidor para aceptar **conexiones remotas** (editando el archivo `mysqld.cnf` y cambiando `bind-address` a `0.0.0.0`).

Terminal mostrando la ejecución de `mysql -u chinos_user -p` y las tablas de `chinos_cafe`

def conectar_db():

        print(f"\n❌ ERROR al conectar a MySQL. Revise la IP ({DB_CONFIG['host']}) y credenciales.")
